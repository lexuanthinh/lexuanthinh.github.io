DSSAMA Getting started

In this chapter, we’re going to walk you through using DSSAMA to solve the classification problem of the Titanic Dataset

1. Dataset Description:

Titanic is a famous dataset of classification problem. In this dataset’s table, you are provided with information on 887 passengers, each of them is a table row with the features: - Pclass - Name - Sex (Male/Female) - Age - Siblings/Spouse Abroad - Parents/Children - Fare - Survived (in which value 1 denoting survival, 0 denoting death) Your task is to build a classification model, using the input features Pclass, Name, Sex (Male/Female), Age, Siblings/Spouse Abroad, Parents/Children, Fare to predict the output (Survival) of the passenger.

Titanic Dataset - train.csv

2. Logging in and creating Workspaces on DSSAMA

In order to use the DSSAMA platform, first sign up and create your account at DSSAMA. After you sign in, create a new account by clicking the plus button, after which a workspace window will appear as illustrated below. Then, fill in the information and click “save”. The process in described in the picture below. For the server option you can either use default server (free), or paid option. Please be noted that you can only create one workspace with the default workspace server. Any and all subsequent workspaces will have to use other server.

For each workspace you can create multiple project corresponding to multiple tasks. Now, we will create a project for the Titanic Classification problem.

Scroll down and you will see the Data Source path, Datasource is the part where you add the datasets for your projects. Click “Create new datasource” in the section “List of data source”, fill in the required fields of information for the Titanic dataset. • Data source: As this dataset is in tabular format, we click “Tabular” • Source type: Choose Csv file. • Choose File: upload the “train.csv” file of the Kaggle dataset that we downloaded earilier. • ID column: Select the column that contain data’s ID. In this file, that would be *PassengerId • Label column: Choose the column that contains the data’s label. In this problem, we select the column Survived

After you click “Save”, datasource can be saved to used. - Flows is where we work directly with the datasets. For each problem, we can create a corresponding flow.

3. Building flow for the problem

  • Dataflow Editor: the grid area to create and adjust working flow.
  • Stage Repository: List of available Stages on the platform, Stages fall into three categories, can be the Input Data format, data processing techniques, or machine learning algorithms, organized in a tree directory.
  • Stage Properties: The properties for each Stage, how you adjust the Stage Properties will affect the result of the problem. For example:
  • Console Logs: Show the execution results after you run the flow. Stage and Flow

Stage và Flow

  • On the DSSAMA, data-processing tasks are carried out individually in Stages.
  • For each stage once in the Flow Editor Grid, they are represented as squared blocks and can be sequentially linked to each other by arrows, each stage will take the output of the previous stage as input (except for the Input Stage), and in return, its output will be the next stage’s input. Once you set up the Stage, click “Run” on the toolbar to execute.
  • After we finish, the final result will be displayed in the console logs and on the dashboard. Each finished Stage will be marked with a green tick on the lower-right corner of their block.

Import data

To import data from the data sources, we use Input Stage. Depending on the data format, we will have the respective Input Stage: • Tabular Input: Input Stage for tabular data type. • Text Input: Input Stage for text data type. • Timeseries Input: Input Stage for time-series data type. • Image Input: Input Stage for image data type. Other miscellaneous Input Stage types: Image Input for Classification, Image Input for Segmentation for other image data source/cho các nguồn dữ liệu dạng ảnh khác. For the Titanic problem, we selected the data source type as Tabular, so to import the dataset, we use Stage Tabular Input. Click on the Stage Tabular Input in the Stage Repository, move it to the Dataflow Editor. Building Flow for the Classification problem with Titanic dataset

1. Tabular Input:

After creating the Classification Flow, in the Flow’s screen Click on the Stage Tabular Input and put it in Dataflow Editor, adjust the properties: • Dataset: select Titanic datasets. • Selected Columns: select the columns that you will use for the problem-solving process. Here we select the column: PClass, Sex, Age, SibSp, Parch. You can use all the columns for the probems. • Click on the Input Stage’s block again and click Run

2. Handling mising data:

Since column Age has 30% missing data, we need to handle it. Choose Stage Simple Imputer Transformer in Transformer Sub-package của Scikitsama.ml and put it in the Dataflow Editor. Link Tabular Input Stage to the Stage Simple Imputer Transformer. Set the properties to handle missing data as shown in the figure below: - Selected column: Select the column to impute. Be noted that we should only choose one column for each Stage Simple Imputer Transformer. Select Age - Missing Values: Select either np.nan hoặc None, depending on the datatype of the missing data. Normally, missing data. Normally, missing data is encoded as blank spaces, NaN or other characters. In DSSAMA, missing data is displayed as either (None) or NaN. - Strategy: Choose the missing-data-handling method from the below: - mean: Fill missing data with the column’s average value. - median: Fill missing data with the column’s median value. - most_frequent: Fill missing data with the column’s most frequently occuring value. - constant: Fill missing data with a constant value, as defined in Fill Value text box. - Fill Value: only activated with constant option in Strategy Once you finish set up, execute the block. For datasource with multiple incomplete columns, we handle them with multiple Simple Imputer Stages. In this dataset, Sex column also has missing values, so we handle it with a new Stage Simple Imputer.

3. String data encoding:

Sex Column contains two values: Male và Female, which can’t be used to put in a ML model, so it needs encoding. Select One Hot Encoder Transformer Stage in Transformer subpackage of Scikitsama.ml. Link them with the previous Stages, select Sex Column and set the properties. After that, click on that Stage and click Run. Besides One Hot Encoder Transformer we could also use other encoding methods in the same sub-package such as Ordinal Encoder Transformer.

4. Data Normalization:

In our current problem, features are in different value ranges, which will make it difficult for the machine learning model to learn. To prevent that, we need to normalize data. In this case, we will normalize Age column. Select Normalize Transformer in Transformer sub-package of Scikitsama.ml, link it to the previous Stage (One Hot Encoder Transformer). Select colum Age, set the property click again on the Stage and click Run.
